# Skill: make-pr

## Purpose

Create a GitHub pull request from the current working tree. Handles the full workflow: selective file staging, conventional commit, branch creation, push, and PR creation with an auto-generated summary.

## Invocation

```
/make-pr
/make-pr fix: resolve chat scroll bug
/make-pr --draft
```

**Examples**:
- `/make-pr` - Auto-infer title from changes, create PR
- `/make-pr feat: add lease export` - Create PR with explicit title
- `/make-pr --draft` - Create as draft PR

## Inputs

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `title` | No | Inferred | PR title (conventional commit format) |
| `type` | No | Inferred | One of: feat, fix, chore, refactor, docs, style, test |
| `--draft` | No | false | Create as draft PR |
| `--base` | No | main | Base branch for the PR |
| `--body` | No | Auto-generated | Custom PR body text |

## Input Validation

Before execution, validate:
1. `gh` CLI is authenticated (`gh auth status`)
2. Git repo is clean or has changes to commit
3. Remote `origin` is configured

## Execution Steps

### 1. Pre-Flight Checks

```bash
# Verify gh CLI
gh auth status

# Check current branch and status
git branch --show-current
git status --porcelain
```

- If no changes and no unpushed commits, report "Nothing to PR" and stop.

### 2. Branch Creation (if on main)

If currently on `main`:
- Determine `type` from the title or ask the user
- Generate branch name: `<type>/<kebab-case-description>`
- Create and switch: `git checkout -b <branch>`

Branch naming conventions:

| Prefix | Purpose |
|--------|---------|
| `feature/` | New functionality |
| `fix/` | Bug fixes |
| `chore/` | Maintenance |
| `docs/` | Documentation |
| `refactor/` | Code restructure |

### 3. Selective Staging (CRITICAL)

**NEVER run `git add .` or `git add -A`.** Unrelated local changes frequently exist from worktrees or parallel work.

```bash
# List all changed files
git status --porcelain
git diff --stat
```

**Process:**
1. Display the full list of changed/untracked files to the user
2. Identify which files are relevant to this PR's purpose
3. If ANY files look unrelated to the task, explicitly call them out and ask the user to confirm
4. Stage ONLY the approved files: `git add <file1> <file2> ...`

**Warning signs of unrelated changes:**
- Files in directories unrelated to the PR topic
- Config files that weren't intentionally modified
- Files from other feature branches or worktrees
- Untracked files that look like scratch/temp work

### 4. Commit

Generate a conventional commit message:

```bash
git commit -m "<type>: <description>"
```

Format:
- Present tense ("add feature" not "added feature")
- Lowercase after type prefix
- One logical change per commit

### 5. Push Branch

```bash
git push -u origin <branch-name>
```

### 6. Create PR

Auto-generate the PR body by summarizing the diff:

```bash
# Get diff stats for body generation
git diff main --stat
git diff main
```

Then create the PR:

```bash
gh pr create \
  --title "<type>: <description>" \
  --body "<auto-generated summary>" \
  --base main \
  [--draft]
```

**Auto-generated body format:**

```markdown
## Summary

<Brief description of what changed and why>

## Changes

<File-by-file summary of modifications>

## Testing

<How to verify these changes>
```

### 7. Post-Creation

- Report the PR URL and number
- Show CI status link
- Suggest self-review if a review-pr skill exists

## Output

```markdown
## PR Created: #48 - feat: add lease export endpoint

**URL**: https://github.com/org/repo/pull/48
**Branch**: feature/lease-export -> main
**Status**: Open

### Files Included
- src/api/export/route.ts (+120)
- src/lib/export-utils.ts (+45)

### Files Excluded (unrelated)
- .mcp.json (local config)
- scripts/debug.py (unrelated script)

### Next Steps
- CI will run automatically
```

## Error Handling

| Error | Action |
|-------|--------|
| No changes to commit | Report "Nothing to PR", suggest what to check |
| Branch already has open PR | Show existing PR URL and number |
| Push rejected | Report error, suggest `git pull --rebase` |
| gh auth failure | Report error, suggest `gh auth login` |
| On detached HEAD | Report error, suggest checking out a branch |
| Merge conflicts with base | Report conflicts, suggest resolving first |

## Dependencies

- `gh` CLI installed and authenticated
- Git repository with `origin` remote configured
- GitHub repository access (push permissions)
